Site Web

Introduction

Afin de contrôler le système et suivre l'état de la plante, une plate-forme est nécessaire. Un choix a du être fait entre une application Android et un site web. J'ai suggéré la création d'une solution hybride, à savoir un site web s'adaptant au mobile grâce au "responsive design" car nos connaissances en développement d'applications mobiles sont très limitées et la création d'un site web était plus abordable.

Configuration et attribution d'un nom de domaine

Tout site web a besoin d'un identifiant de domaine internet, appelé nom de domaine : c'est une adresse Internet alphabétique, constituée d'un nom que vous choisissez et d'une extension séparés par un point. Obtenir un nom de domaine en .fr, .com ou autre extension générique n'est pas gratuit et est contrôlée par un organisme comme l'ICANN pour les extensions génériques ou une institution gouvernementale. Afin d'obtenir un nom de domaine gratuit nous avons utilisé Github, un service web d'hébergement et de gestion de développement de logiciels. Ce service offre également la possibilité d'héberger son propre site web, avec comme nom de domaine du type : https://username.github.io . La limite de ce type de site web est la taille du contenu du site web puisqu’elle est limitée à 1Gb. Cependant, cet espace de stockage largement suffisant pour le projet.

La publication du site web fonctionne sur le principe de Github qui permet de travailler en équipe. Chaque participant du projet à son espace de travail propre et "Commit (sauvegarder) " vers son répertoire local Github lorsqu'il apporte une modification à ses fichiers afin de créer une nouvelle version de son projet. Ensuite, les modifications sont "Push (envoyer) " vers le répertoire maître sur les serveurs Github. Les autres utilisateurs, pour récupérer les modifications peuvent "Fetch (aller chercher) " depuis le répertoire maître vers leur propre répertoire local.Le principe est résumé en annexe 4. Pour créer un site web, le fonctionnement est sensiblement le même. Chaque modification sur les fichiers codant le site web ( HTML et CSS ) est commit vers notre répertoire local afin de sauvegarder une nouvelle version mais elle n'est publiée en ligne qu'après "Push"

HTML

HTML , est un language conçu pour concevoir des pages web. Il est caractérisé par son fonctionnement en balisage. Chaque balise donne un caractère spécial au texte. Par exemple: , <body>, <header>, <footer>, <article>, <section>, <p>, <div>, <span>, <img> sont des balises essentielles en \gls{html}. Chaque page \gls{html} débute par les balises de définition et le corps de la page est défini par : <!DOCTYPE html> // Le type de document <html> //element racine <head> // En-tete de la page HTML <meta charset="utf-8"> // jeu de caracteres a utiliser <title>Ma page test // Titre // Fin de balise // Corps de la page

CSS

CSS est un langage en complément du langage HTML visant à coder la mise en forme d'une page web. Une feuille de style, en .CSS se lie au code source d'une page web via l'ajout de la commande suivante dans l'en-tête du code source de la page HTML :\begin{lstlisting}[style=CStyle] \end{lstlisting} Elle permet de caractériser le design d'une classe d’éléments HTML. Iframe

Afin de visualiser les données des capteurs, il est nécessaire d'exporter les graphiques de Thingspeak vers le site web. Cela se fait grâce aux balises IFRAME. Ces balises permettent d'insérer des données provenant d'un autre serveur, ici Thingspeak. Ainsi chaque graphique ou affichage est insérer dans le code HTML.
