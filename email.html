Introduction

Notre plate-forme de commande étant un site web , il est impossible de créer des notifications afin d'alerter l'utilisateur concernant l'état de santé de la plante. Ainsi, la solution retenue est la suivante : créer un système de notifications par mail dès que certaines conditions sur les données Thingspeak sont atteintes. Les alertes suivantes sont envoyées :

  • Si la luminosité pendant plusieurs jours est trop importante ou insuffisante.
  • Si la température pendant plusieurs jours est trop importante ou insuffisante.
  • Dès que la pompe est activée.
  • Lorsque le niveau d'eau dans le reservoir est trop faible .

Configuration et attribution d'un nom de domaine

Les alertes sont gérées via IFTTT . C'est un service permettant de créer des chaînes d'instructions simples appelées applets. Une applet est configurée pour effectuer une action lorsque qu'un changement est effectué sur une plate-forme, comme Thingspeak par exemple. L'applet est configurée pour envoyer un email lorsque qu'elles sont sollicitées. Les applets sont définies par leur ID et sont sollicitéS dans Thingspeak via un module Matlab et TimeControl. Le TimeControl permet d'enclencher une \gls{api} Thingspeak à un intervalle de temps défini. Par exemple, l'alerte email concernant le taux d'eau dans le réservoir est reglé sur 1 fois par heure. Toutes les heures le TimeControl va activer un module Matlab. Le code est défini en annexe 7.
